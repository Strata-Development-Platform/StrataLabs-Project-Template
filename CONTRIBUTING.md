# Contributing

Thank you for helping improve this project.

## Workflow

1. Check existing issues and documentation.
2. Open an issue before substantial behavioral or architectural changes.
3. Fork the repository and create a focused branch.
4. Add or update tests and documentation.
5. Run the project's validation commands.
6. Open a pull request using the provided template.

Keep `.stratalabs/project.json` and `.stratalabs/roadmap.json` valid JSON. Roadmap changes should explain user impact in the pull request.
